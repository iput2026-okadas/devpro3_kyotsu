# CSV ビューア E2E テストシナリオ

Playwright の Chromium から Flask 製CSVビューアへアクセスして確認します。

## 1. 最新CSVの初期表示

- CSVビューアのトップページを開く。
- 一覧にテスト用CSVが2件表示されることを確認する。
- 最新のCSVが選択状態になることを確認する。
- CSVの7列、3行、表示件数が正しく表示されることを確認する。
- `client_id` 列に各行の送信元Raspberry Piが表示されることを確認する。

## 2. CSVファイルの切り替え

- トップページを開く。
- 一覧から古いCSVのリンクをクリックする。
- URLと選択状態が古いCSVへ切り替わることを確認する。
- 古いCSVの2行が表示され、IDが `101`, `102` であることを確認する。

## 3. CSV平均値の表示と再計算

- トップページを開く。
- 最新CSVの平均温度が `19.8 ℃`、平均湿度が `47.8 %` と表示されることを確認する。
- 平均CO2が `1116.7 ppm`、平均光量が `48.3 %` と表示されることを確認する。
- 一覧から古いCSVへ切り替える。
- 平均温度が `22.8 ℃`、平均湿度が `45.8 %` に更新されることを確認する。
- 平均CO2と平均光量も選択したCSVの値へ更新されることを確認する。

## 4. 数値列の並べ替え

- トップページを開く。
- `temp` 列の見出しを1回クリックする。
- 気温が数値として昇順に並び、昇順アイコンが表示されることを確認する。
- `temp` 列の見出しをもう1回クリックする。
- 気温が降順に並び、降順アイコンが表示されることを確認する。

## 5. CSV・JSONエクスポート

- トップページを開く。
- CSVエクスポートボタンをクリックする。
- 選択中のCSV名で `client_id`、`co2`、`light_percent` を含む7列、4行のデータが
  ダウンロードされることを
  確認する。
- JSONエクスポートボタンをクリックする。
- 選択中のCSV名に対応するJSONがダウンロードされ、IDとクライアントIDが表示順で
  含まれることを確認する。

## 6. センサーデータの追加

- 「データ追加」をクリックし、専用ダイアログが表示されることを確認する。
- CSVに存在する `client_id` を選択でき、timestampの初期値が秒まで設定されることを
  確認する。
- timestamp、temp、humid、co2、light_percentを入力して追加する。
- 再表示後もco2とlight_percentの列および値が表示されることを確認する。
- 追加行に次のレコードIDが採番され、timestamp順の位置に表示されることを確認する。
- 入力したすべての値と表示件数が正しいことを
  確認する。

## 7. データ追加のキャンセル

- 「データ追加」をクリックして専用ダイアログを開く。
- 「キャンセル」をクリックするとダイアログが閉じることを確認する。
- CSVのデータ行数が変わらないことを確認する。

## 8. 存在しないCSVの指定

- URLの `file` パラメーターへ存在しないCSV名を指定する。
- ファイルが見つからない旨のエラーメッセージが表示されることを確認する。
- データ行が表示されないことを確認する。

## テストデータと実行環境

- `tests/fixtures/csv_viewer/` のCSVを一時ディレクトリへコピーして使用する。
- `Server/data/` のサンプルデータは変更しない。
- FlaskサーバーとChromiumはテスト中に自動起動し、終了時に停止する。
- 画面を見ながら確認する場合は `E2E_HEADLESS=false` と `E2E_SLOW_MO` を指定する。
- 詳細な実行コマンドは `tests/README.md` を参照する。
